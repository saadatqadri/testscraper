3.2.3 Reporting data changes to EMP

3.2.3.1 Status Point Processing

3.2.3.1.1 Status Point Alarm

Any status point state can be marked to indicate that uncommanded transitions into the state are either Alarmable or not Alarmable. Commanded changes are never alarmed. This test demonstrates that when the status point undergoes a transition, the point is correctly presented.

Estimated time:         30-40 minutes

3.2.3.1.2 Commanded Trip

Setup:            Using the SCADA device display, choose a status point that is connected to a test fixture, that is not auto acknowledged,(Check the auto ack flag on the point record) and that has alarm states defined. The Alarmable states for each point-type are defined on the SCADA  Tabular Display in the SMP application. The chosen point should be of GOOD data quality, it must have usable controls defined, alarms must not be inhibited, and the current state of the point should be CLOSED. It must be a point in which the RTU can detect momentary changes between scans.

Proced.: Command a change of state to TRIP

Results: The point will not be alarmed on transition into the state.

Verif.: Check for the alarm, there should be none. Check that the commanded state change has taken place and that the log indicates that the breaker has tripped.

Clean-Up: None. This test is the setup for the test in the following section.

3.2.3.1.3  Uncommanded Close

Setup:            None. The previous test is the setup for this test.

Proced.: On the test fixture, toggle the test point back to the CLOSED state.

Results: the point will be alarmed on the transition into the state.

Verif.: Check for the alarm. Check that the uncommanded state change has taken place and that the log indicates that the breaker has closed.

Clean-Up: Acknowledge the point. This test is the setup for the test in the following section.

3.2.3.1.4 Uncommanded Trip

Setup:  None.  The previous test is the setup for this test.

Proced.: On the Test Fixture, toggle the test point to TRIP.

Results: The point will be alarmed on the transition into the state.

Verif.:     Check for the alarm.  Check that the uncommanded state change has taken place and that the log indicates that the breaker has tripped.

Cleanup: Acknowledge the point.  This test is the setup for the test in the following section.

 

3.2.3.1.5 Commanded Close

Setup:  None.  The previous test is the setup for this test.

Proced.: Command a change of state to CLOSE.  This can be done from the EMP Substn Tabular display.

Results: The point will not be alarmed on the transition into the state.

Verif.:     Check for the alarm.  There should be none.  Check that the state change has taken place and that the log indicates that the breaker has closed.

Cleanup: None.  This test is the setup for the test in the following section.

3.2.3.1.6  Uncommanded Reclosure

Setup:  None.  The previous test is the setup for this test.

Proced.: On the Test Fixture, toggle the point to TRIP momentarily, then return it to CLOSE.  This two step transition must be accomplished between two scans.

Results: The momentary change will be alarmed.

Verif.:     Check for the alarm.  Check that the state remained CLOSED and that a message in the System Activity Log indicates that the breaker has tripped and closed.

Cleanup: Acknowledge the reclosure.  This test is the setup for the test in the following section.

3.2.3.1.7 Uncommanded failed reclosure

Setup:  None.  The previous test is the setup for this test.

Proced.:

  1. Disable the Scan Group from the RTUSTATE display.
  2. On the Test Fixture, perform a TRIP
  3. Perform a CLOSE
  4. Perform a TRIP
  5. Enable the Scan Group

Results: The point will be alarmed on the transition into the state.

Verif.:     Check for the alarms.  Check that the state change has taken place and that a single message in the log indicates that the breaker has tripped, closed, and tripped (Close Trip Close Trip).

Cleanup: Acknowledge the point.  This test is the setup for the test in the following section.

3.2.3.1.8 Other Uncommanded Momentary Changes

Setup:  None.  The previous test is the setup for this test.

Proced.: On the Test Fixture, toggle the point to CLOSE momentarily, then back to TRIP.  All this must be completed between two scans.

Results: The point will be alarmed on the transition into the state.

Verif.:     Check for the alarms.  Check that the state change has taken place and that a single message in the log indicates that the breaker has both closed and tripped (Trip Close Trip).

Cleanup: None.  This test is the setup for the test in the following section.

3.2.3.1.9 Acknowledge Status Point Alarm

This section demonstrates that alarms on a status point can be acknowledged.

 

Setup:   The test in the previous section is the setup for this test.

Proced.: From the Scada tabular display, acknowledge the alarm on the status point from the previous section.

Results: The alarm on the point will be acknowledged.

Verif.: verify that the point's UNACK bit is reset.

Cleanup: None.

3.2.3.2 3-State Status Point Processing

Estimated time:         10-15 minutes

3.2.3.2.1 Uncommanded Operation

This section tests, alarming of an uncommanded operation of a three-state device.

Setup:   Using the Scada Tabular display, choose a 3-state status point that is connected to the test fixture and has alarmable and illegal states defined.  In fact, all legal states should be alarmable.  The point should be of GOOD data quality, alarms for the point must not be inhibited and its current state should be CLOSED.  Be sure that the control time-out is long enough to allow manual flipping of the associated status bits when commanded to change.  Refer to the CONV record in the SCADA modeling application to obtain the meaning of the chosen 3-state point's four combinations for tests in this section.

Proced.: On the Test Fixture, toggle the switches for the point to represent the OPEN state.

Results: The point will indicate OPEN and will be alarmed.

Verif.:     Check for the alarm.  Check that the uncommanded state change has taken place and that the log indicates that the breaker has tripped.
